# Synthesis
This directory contains the two parallel synthesis flows maintained throughout the project. Both target the same standard cell library via Design Compiler, but diverge in their treatment of the instruction memory.
The Questa flow (`/Questa/`) synthesizes `ins_mem.v` as a black box, allowing the module to load instructions at runtime from a `.bin` file generated by the Python assembler (`../assembler/assembler.py`). This preserves full flexibility for waveform verification — any assembly program can be simulated without resynthesis.
The Innovus flow (`/Innovus/`) replaces the instruction memory with a hardcoded case statement (`hardcode_ins_mem.v`), synthesizing the instruction program directly into a gate-level MUX tree. This is required for physical design since Innovus cannot place and route a black box module. The `run` script in each folder reproduces the synthesis and simulation flow end to end.
